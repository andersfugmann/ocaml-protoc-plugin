Module Protobuf.Value

`Value` represents a dynamically typed value which can be either null, a number, a string, a boolean, a recursive struct value, or a list of values. A producer of value is expected to set one of these variants. Absence of any variant indicates an error.

The JSON representation for `Value` is JSON value.

type t = [
  1. | `not_set
  2. | `Null_value of NullValue.t
  3. | `Number_value of float
  4. | `String_value of string
  5. | `Bool_value of bool
  6. | `Struct_value of Struct.t
  7. | `List_value of ListValue.t
]

The kind of value.

`Null_value Represents a null value.

`Number_value Represents a double value.

`String_value Represents a string value.

`Bool_value Represents a boolean value.

`Struct_value Represents a structured value.

`List_value Represents a repeated `Value`.
